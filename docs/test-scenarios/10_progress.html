🔳 GAP-005 — Automated GPA Threshold Alert System

What's missing: When a student's GPA drops below a configurable threshold (2.0 academic probation, 2.5 scholarship, 1.8 withdrawal warning), the system does not automatically generate an alert, notify the advisor, or update the student's dashboard banner.

Root cause: The Proactive Intelligence batch (POST /api/ai/proactive-intelligence) does check EdNex flags but does not query mod01_student_profiles.cumulative_gpa against threshold rules. The threshold logic was planned in Phase 3 (Predictive Analytics) but not yet implemented.

To implement:

  • Add a gpa_threshold_check() function that compares cumulative_gpa against a rules table
  • Insert a row into mod03_intervention_flags with flag_type="GPA_BELOW_THRESHOLD"
  • Wire the flag to the student's Holds & Alerts view
  • Send an advisor notification via the existing notification channel
  • Estimated effort: ~4 hours backend + 2 hours frontend integration
